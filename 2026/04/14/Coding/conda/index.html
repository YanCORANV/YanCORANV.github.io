对于需要处理 SMPL 模型、PyTorch 渲染、CUDA 加速的研究项目，Conda 是事实上的标准

1. Conda 与 .venv 的本质区别

特性 .venv Conda
隔离层级 仅限 Python 库 Python 版本 + 库 + 二进制依赖 (C++/CUDA)
存储方式 项目内路径 (如 Blog/.venv) 集中管理 (如 ~/miniconda3/envs/)
包来源 PyPI (pip) Conda Channels (conda, conda-forge)

2. 为什么机器人研究必用 Conda？

在处理 VLA 模型或 SMPL-X 时，你经常会遇到以下需求：

  • Python 版本切换：有些旧的代码库必须在 Python 3.8 运行，而新模型需要 3.12。Conda 可以轻松创建不同版本的 Python 环境
  • CUDA 依赖：Conda 可以为每个环境安装不同版本的 cudatoolkit，而不需要你频繁修改系统的环境变量

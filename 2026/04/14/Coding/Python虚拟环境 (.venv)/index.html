当你运行 pip install 时，默认会将包安装到全局 Python 目录中。这会导致“依赖冲突”

1. 为什么需要 .venv？

假设项目 A 需要 PyYAML 5.0，项目 B 需要 PyYAML 6.0。在全局环境下，二者无法并存。.venv 会在你的项目目录下创建一个独立的文件夹，其中包含：

  • Python 解释器的副本
  • 独立的 site-packages 文件夹

2. .venv 的局限性

.venv 只能管理 Python 库。它无法处理非 Python 的依赖项（例如特定版本的 C++ 编译器、CUDA 驱动、或是 ffmpeg 等系统工具）。这正是 ECE 领域的机器人研究最痛苦的地方
